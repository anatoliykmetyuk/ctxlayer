For engineers who use AI daily. Accumulate and link domain knowledge to your project via symlinks.

Quick start

Run ctx new to create a new domain and a new task within that domain. Tasks live within domains. They live system-wide in ~/.agents/ctxlayer/projects/.

What you can do after ctx new

  • Put reference material in data/, docs and learnings in docs/
  • Ask your agent: “in the context layer, read document X”
  • Ask your agent: “Please write a report of what you just did to the context layer”
  • Use ctx import to link tasks and intelligence from other projects — reuse knowledge across projects
  • Your developer-specific intelligence stays with you, separate from the common repo — it never pollutes version control

How it works

Intended algorithm

For continuous AI-assisted development. Six steps:

  1. Prepare environment — ctx new to create a task
  2. Collect intelligence — Error messages, issues, sample data, prior work
  3. Compile a plan — Describe the task, produce a plan (AI-assisted)
  4. Iterate on the plan — Review, adjust, refine
  5. Execute — Ask the agent to implement
  6. Log the knowledge — Ask AI to compile learnings into markdown in docs/

Principles

  • Garbage in, garbage out — Collect good data and articulate what you want.
  • AI strengths — Good at info collection and implementation; weaker at architecture and high-level decisions. Give it clear concepts and data.
  • Core idea — Collect data continuously as work progresses.
